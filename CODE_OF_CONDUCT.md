# Code of Conduct

## Our Standard

notifycat should be a practical and welcoming project for maintainers, users,
and contributors. We expect participants to act with respect, assume good
intent, and focus disagreement on ideas and tradeoffs rather than people.

Examples of encouraged behavior:

- Use clear, considerate language.
- Accept constructive feedback.
- Keep discussions on topic and useful to the project.
- Respect different experience levels and backgrounds.
- Give credit when building on someone else's work.

Examples of unacceptable behavior:

- Harassment, intimidation, threats, or sustained disruption.
- Insults, personal attacks, or demeaning comments.
- Sexualized language or imagery.
- Publishing private information without explicit permission.
- Any conduct that would reasonably make participation unsafe or unwelcome.

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, or block
participants when behavior violates this code of conduct.

If you need to report a problem, contact the maintainers through the repository
owner's GitHub profile or a private channel published by the project. Include
links, screenshots, or other context when available. Reports will be handled as
privately as practical.

Maintainers are expected to apply this code consistently and fairly.
